Module Azimuth

Basic module

  • Saving & loading config files
  • Logging functions with separate console and file log levels
  • Ordered 'pairs' function
  • 'serialize' function for debugging tables and serializing data
  • Validation functions for int, float and table values

Usage:

    local Azimuth = include("azimuthlib-basic")

getFloat (value, bounds[, default[, isBoundsEnum=false]])
Validation function that checks if a value is not empty, correct type and within borders

Parameters:

  • var value — Any variable for checking
  • table bounds — 2-element table with lower and upper bounds. If isBoundsEnum is true however, it will be considered a table that contains all allowed values
  • number default (optional) — If nil, incorrect value with result in nil. If number, incorrect value will return specified default value
  • number isBoundsEnum (default: false) — Signals that bounds is a enum of allowed values

Returns:

  1. nil/float result
  2. bool isModified — If true, value was incorrect in some way

Usage:

  • -- if incorrect returns nil. if outside of bounds returns closes bound
local result = Azimuth.getFloat(value, {4.4, 15})
  • -- if incorrect returns default value (5.3). if outside of bounds, returns closest bound
local result = Azimuth.getFloat(value, {4.4, 15}, 5.3)
  • -- bounds are treated like enum instead. if incorrect/doesn't match enum, returns default (5.3)
local result, isModified = Azimuth.getFloat(value, {4.4, 5.3, 9.9}, 5.3, true)
if isModified then
    print("passed value was originally invalid")
end

loadConfig (modName, options[, isSeedDependant=false[, modFolder=false]])
Loads mod config from file

Parameters:

  • string modName — Mod name (file name)
  • table options — Config options with default values (key [1] or ["default"]) and other optional properties. More about config options: config-options.lua
  • bool isSeedDependant (default: false) — Set to true if config is specific for this server (it's useful only on client side)
  • bool/string modFolder (default: false) — If true, then config will be loaded from "moddata/ModName/ModName.lua". Or you can specify different folder name

Returns:

  1. table modConfig — Loaded mod config table
  2. var

    status — Error/status. Can be one of the followingː

    • string — It's an error message.
    • number, 1 — File wasn't found. It's a hint that you should re-save the config.
    • number, 0 — Config was successfully loaded, but was modified by options (maybe some variable didn't fit in the boundaries). It's a hint that you should re-save the config.
    • nil — Config was successfully loaded, no modifications were made.

Usage:

  • local tbl = Azimuth.loadConfig("MyMod", { WindowWidth = {300} })
  • local configOptions = {
  WindowWidth = {300, round = 1, min = 100, max = 600, comment = "UI window width"}
}
local modConfig, isModified = Azimuth.loadConfig("MainConfigFile", configOptions, true, "MyMod")
if isModified then
    Azimuth.saveConfig("MainConfigFile", modConfig, configOptions, true, "MyMod")
end

saveConfig (modName, config[, options[, isSeedDependant=false[, modFolder=false[, minify=false]]]])
Saves mod config to file

Parameters:

  • string modName — Mod name (file name)
  • table config — Mod config table
  • table options (optional)

    — Config options with default values and comments. Each element of the table can have following propertiesː

    • var [1] (optional) — Default value of a variable (new format). Will be added to a commentary unless it's a table
    • var default (optional) — Default value of a variable (old format). Will be added to a commentary unless it's a table
    • string comment (optional) — Variable description
  • bool isSeedDependant (default: false) — Set to true if config is specific for this server (it's useful only on client side)
  • bool/string modFolder (default: false) — If true, then config will be loaded from "moddata/ModName/ModName.lua". Or you can specify different folder name
  • bool minify (default: false) — If true, strips saved config of all spaces and new lines

Returns:

  1. bool success
  2. string/nil error — Explains what went wrong

Usage:

    local configOptions = {
  WindowWidth = {300, min = 100, max = 600, round = -1, comment = "UI window width"}
}
Azimuth.saveConfig("MainConfigFile", modConfig, configOptions, false, "MyMod")
serialize (o[, options[, prefix=""[, addCarriageReturn=false[, minify=false[, recursive=false]]]]])
Serializes table as readable multi-line text

Parameters:

  • table o — Table for serialization
  • table options (optional) — Since this function was initially meant to aid in saving config files, you can add default value and commentary to the each top-level variable
  • string prefix (default: "") — Line prefix, used by the function itself (default: "")
  • bool addCarriageReturn (default: false) — If true, function will use "\r\n" as new line instead of "\n". False by default because "\r\n" messes up Avorion file logs
  • bool minify (default: false) — If true, the resulting string won't have spaces and line breaks. Disables 'prefix' and 'options'
  • bool recursive (default: false) — If true, the resulting table dump will have "__address" field for each non-empty table, containing its address

Returns:

    string result — Serialized table

Usage:

  • print(Azimuth.serialize(myTable))
  • print(Azimuth.serialize({myVar = 30}, {myVar = {default = 20, comment = "This variable does stuff"}}))
  • print(Azimuth.serialize({myVar = 30}, {["myVar"] = {20, comment = "This variable does stuff" }}))
